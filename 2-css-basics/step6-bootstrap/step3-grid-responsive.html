📌 패턴 2: 메인-서브 구조

💡 사용 시나리오: 기사 본문 + 사이드바, 상품 상세 + 관련 상품
🎯 장점: 정보 위계 명확, 주요 콘텐츠 강조

💡 사용 시나리오: 팀 소개, 서비스 특징, 포트폴리오 갤러리

프로젝트에서 바로 써먹는 노하우

✅ 좋은 관례 (Best Practices)
  • 📱 Mobile First 설계
    가장 제약이 많은 환경부터 시작
  • 🎯 핵심 브레이크포인트만 사용
    보통 SM, MD, LG 정도면 충분
  • 📊 실제 사용자 데이터 기반 결정
    구글 애널리틱스 기기 분석 활용
  • 🔄 점진적 향상 (Progressive Enhancement)
    기본 기능 → 고급 기능 순서
  • 🧪 실기기 테스트
    개발자 도구 + 실제 디바이스 병행
❌ 피해야 할 실수들
  • 🚫 Desktop First 설계
    모바일에서 사용하기 어려운 결과
  • 🚫 모든 브레이크포인트 정의
    복잡성만 증가, 유지보수 어려움
  • 🚫 콘텐츠 우선순위 무시
    모바일에서 중요한 정보가 아래쪽에
  • 🚫 터치 영역 크기 간과
    버튼이나 링크가 너무 작아서 클릭 어려움
  • 🚫 성능 최적화 무시
    모바일 데이터 환경 고려하지 않음

📱 반응형 설계 원칙
  • Mobile First: 작은 화면부터 설계
  • 점진적 향상: 큰 화면에 기능 추가
  • 콘텐츠 우선순위: 중요한 정보 먼저
  • 사용자 행동: 각 기기별 사용 패턴 고려
🔧 실무 체크리스트
  • 브레이크포인트: 2-3개만 핵심적으로
  • 터치 영역: 최소 44px 이상
  • 테스트: 실제 디바이스에서 확인
  • 성능: 모바일 데이터 환경 고려
